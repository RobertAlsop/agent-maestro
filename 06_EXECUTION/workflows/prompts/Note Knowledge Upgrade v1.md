We are now performing a focused knowledge upgrade for notes inside the /AGENT_MAESTRO/ folder in my Obsidian Vault.

The goal is to strengthen definitions and improve the ontology without changing the conceptual structure of the vault.

You will work on ONE folder at a time.

Target folder:
<INSERT FOLDER PATH HERE>

Example:
02_KNOWLEDGE/concepts
02_KNOWLEDGE/domains
02_KNOWLEDGE/patterns
etc.

Before editing any note, read all notes in the target folder to understand the conceptual relationships between them.

Then upgrade each note according to the rules below.

--------------------------------------------------

PRIMARY GOAL

Improve conceptual clarity, definitions, and internal linking while preserving the existing ontology.

This is an editorial upgrade, not a rewrite.

--------------------------------------------------

STRICT RULES

1. Do NOT change frontmatter fields.
2. Do NOT change the note ID.
3. Do NOT change filenames.
4. Do NOT change folder structure.
5. Do NOT introduce new concepts unless absolutely necessary.
6. Do NOT rename existing concepts.
7. Do NOT remove existing content unless it is clearly redundant.

--------------------------------------------------

WHAT YOU SHOULD IMPROVE

For each note:

1. Strengthen the core definition

Ensure the opening definition clearly answers:

"What is this concept in the Agent Maestro system?"

2. Clarify boundaries

If useful, briefly clarify:
- what this concept includes
- what it does NOT include

3. Improve internal links

Add links to related notes when appropriate.

Prefer linking to:
- concepts
- patterns
- systems

Do NOT over-link.

4. Improve conceptual precision

Replace vague language with precise operational language where possible.

5. Add short examples where helpful

Examples should be short and illustrate how the concept appears in practice.

6. Maintain concise notes

Notes should remain compact.

Target size:

150–400 words per note.

Avoid essays.

--------------------------------------------------

WHAT YOU SHOULD NOT DO

Do NOT:

- expand notes excessively
- introduce speculative ideas
- invent new terminology
- change the ontology
- rewrite the whole note unnecessarily

This is a refinement pass.

--------------------------------------------------

MAP INTEGRITY

Ensure that each note still links to its canonical map.

If additional map links are appropriate, they may be added.

Do not remove map links.

--------------------------------------------------

OUTPUT PROCESS

Step 1

Output a short plan including:

• number of notes in the folder  
• list of notes to be upgraded  
• any ontology risks detected

Wait for approval.

Step 2

Upgrade the notes one by one.

--------------------------------------------------

QUALITY STANDARD

Write like an editor of a technical discipline.

Definitions should feel like entries in a field manual, not blog posts.

Precision and clarity are more important than length.

--------------------------------------------------
