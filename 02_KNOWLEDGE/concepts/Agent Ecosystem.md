# Agent Ecosystem

## Definition

The full organisational deployment of agent systems — the complete set of Agent Constellation|Agent Constellations]], [[Agent System|Agent Systems]], Shared Services]], and governance infrastructure operating within a single organisational context.

## Boundaries

An Agent Ecosystem is broader than any individual Agent Constellation]] or [[Agent System]]. It is the organisational-level view. An ecosystem is not defined by a single workflow or function — it represents the entire agent operating environment of an organisation or a major division.

## Purpose

The ecosystem concept allows Agent Maestro to reason about agent deployments at an organisational scale, including cross-constellation dependencies, shared services, governance consistency, and the relationship between different agent domains operating in the same environment.

## Key Properties

- **Organisational scope** — spans multiple functions, departments, or domains
- **Governed** — operates under a consistent Authority Architecture]]
- **Composed of constellations** — each major operational domain has one or more constellations
- **Shared services** — uses Shared Services]] for reusable capabilities (memory, orchestration, governance)

## Relationships

- Agent Constellation]] — the primary structural unit within an ecosystem
- [[Agent System]] — individual capability-delivering systems, part of constellations
- Shared Services]] — reusable services consumed across the ecosystem
