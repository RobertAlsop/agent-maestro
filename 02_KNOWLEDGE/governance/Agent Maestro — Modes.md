# Agent Maestro — Modes

## Purpose

Define the five operational modes that structure Agent Maestro work.

## Principle

Every Agent Maestro activity belongs to one of five modes. Modes prevent context-mixing and keep work purposeful.

## Modes

| Mode | Focus |
|---|---|
| **Exploration** | Discover opportunities for operational intelligence |
| **Architecture** | Design the structure of agent systems |
| **Deterministic** | Define contracts, validation rules, and authority boundaries |
| **Execution** | Deploy and operate systems |
| **Evolution** | Improve systems through feedback loops and operational learning |

## Implications

- Work must be tagged to a mode.
- Mixing modes in a single session leads to architectural drift.
- [[Controlled Evolution Pattern]] depends on moving through modes in order.

## When It Applies

Use to orient every working session and every project phase.
