# Agent Maestro — Modes

## Purpose

Define the six operational modes that structure Agent Maestro work. Modes are aligned with the universal pipeline — each mode corresponds to a pipeline stage and a stage of the Irreducible Loop.

## Principle

Every Agent Maestro activity belongs to one of six modes. Modes prevent context-mixing and keep work purposeful. They are the fractal instantiation of the Process seed (Irreducible Loop) applied to project lifecycle.

## Modes

| Mode | Focus | Pipeline Stage | Loop Stage |
|---|---|---|---|
| **Capture** | Receive and gather input — observations, ideas, raw material | 01_CAPTURE (Input) | Perception |
| **Interpret** | Understand, classify, connect — transform input into knowledge | 02_KNOWLEDGE (Interpret) | Interpretation |
| **Decide** | Evaluate options, design solutions, commit to plans | 03_DECIDE (Decide) | Decision |
| **Execute** | Build, implement, create — do the work | 04_EXECUTE (Act) | Action |
| **Record** | Document outcomes, log results, update registries | 05_RECORD (Record) | Memory |
| **Reflect** | Review, learn, propose improvements, brainstorm next steps | 06_REFLECT (Reflect) | Feedback |

## Traceability to Foundations

The six modes are a fractal instance of the Process seed. Each mode corresponds to one stage of the Irreducible Loop:

```
perception → interpretation → [gate] → decision → [gate] → action → [gate] → memory → [gate] → feedback
    |              |                       |                    |                  |                  |
 Capture       Interpret               Decide              Execute            Record            Reflect
```

Gates between modes enforce the Constraint seed — no stage transition without validation.

## Implications

- Work should be tagged to a mode when practical.
- Mixing modes in a single session can lead to drift — but practical work often touches multiple modes in sequence (this is the pipeline working as designed).
- The validation gate (between Interpret and Decide) must be visited before Execute — per the Constraint seed.
- Reflect always feeds back into Capture — this is how the system learns and evolves.

## When It Applies

Use to orient working sessions and project phases. When starting work, ask: "Which mode am I in? Which pipeline stage does this serve?"
