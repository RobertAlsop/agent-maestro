# Agent Maestro — Principles

> **Historical reference.** The 12 principles below are now superseded by the foundational kernel and 6 derived principles in [[Agent Maestro — Foundations]]. This document is retained for traceability — each original principle maps to a specific kernel element. See [[AM — Wave 5 — Treatise on Core Principles]] for the full reconciliation.

## Purpose

Twelve principles that originally governed Agent Maestro system design decisions. Now superseded by the Foundations kernel but retained as a reference showing the lineage from individual principles to unified foundational elements.

## Principle

These principles were constraints, not suggestions. They defined the shape of reliable intelligent systems. Their content is now expressed more precisely by the foundational kernel (3 seeds + 1 meta-principle + 3 laws) and 6 derived principles.

## Principles by Tier (Original)

### Tier 1 — Foundation

| # | Principle | Why it was foundational |
|---|---|---|
| 1 | [[Governance Before Autonomy]] | Without this, nothing else in the framework applies. Govern before you scale. |
| 2 | Authority Over Intelligence | The core thesis of Agent Maestro. [[Authority Architecture]] matters more than the model. |
| 7 | [[Human Escalation Pattern]] | Safety boundary. Humans must always hold final authority on irreversible actions. |
| 4 | Transparency by Design | You cannot govern what you cannot observe. All system actions must be observable and auditable. |

### Tier 2 — Architecture

| # | Principle | When it applied |
|---|---|---|
| 3 | [[Deterministic Control Surfaces]] | When defining agent actions and outputs. Actions must be reproducible, traceable, explainable. |
| 5 | Intelligence Through Constraints | When scoping agent authority. Constraints create reliability, not freedom. |
| 6 | [[Three-Layer Memory Model]] | When designing memory and knowledge systems. Three distinct layers, each with clear responsibilities. |
| 8 | Composable Architecture | When structuring multi-agent systems. Systems must remain modular and reusable. |

### Tier 3 — Evolution

| # | Principle | When it applied |
|---|---|---|
| 9 | [[Evidence Over Assumptions]] | When evaluating system behaviour. Behaviour validated through events and outcomes. |
| 10 | Real Systems First | When validating designs. Theory validated in real operational environments only. |
| 11 | [[Controlled Evolution Pattern]] | When upgrading or expanding systems. Evolution through graduated experiments, never jumps. |
| 12 | [[Anti-Agent-Sprawl Pattern]] | When scaling the number of agents. Every agent must have defined purpose, authority, and outcomes. |
