# Cluster 1 — The System-Level Concepts

These define **what the whole domain is about**.

### 1. Operational Intelligence

Definition:

> The system by which organizations execute decisions through structured workflows involving humans and agents.

Everything in the domain ultimately serves this.

Links to:

- Agent Systems
    
- Authority Architecture
    
- System Design
    
- Organizational Operations
    

---

### 2. Agent Systems

Definition:

> Networks of agents coordinated to perform operational tasks.

Links to:

- Agents
    
- Agent Workflows
    
- Orchestration
    
- Operational Intelligence
    

---

### 3. Authority Architecture

Definition:

> The governance system that defines what agents are allowed to do.

This will likely become **the most important node in the domain**.

Links to:

- Authority Packs
    
- Decision Authority
    
- Execution Authority
    
- Memory Authority
    

---

### 4. Memory Architecture

Definition:

> The structured system by which agents store, retrieve, and update knowledge.

Links to:

- Knowledge Graph
    
- Vector Memory
    
- Structured Data
    

---

# Cluster 2 — The Agent Mechanics

These explain **how agent systems actually work**.

---

### 5. Agent

Definition:

> A reasoning entity capable of making decisions and executing actions through tools within defined authority boundaries.

Links to:

- Agent Systems
    
- Agent Workflows
    
- Authority Architecture
    

---

### 6. Agent Workflow

Definition:

> A structured pipeline of tasks and decisions executed by agents and humans.

Links to:

- Operational Intelligence
    
- System Design
    
- Decision Policies
    

---

### 7. Agent Orchestration

Definition:

> The coordination system that manages interactions between agents, tools, and workflows.

Links to:

- Agent Systems
    
- Execution Authority
    
- Workflow Architecture
    

---

# Cluster 3 — The Governance Core

These define **how intelligence is controlled**.

---

### 8. Authority Pack

Definition:

> A portable governance module defining rules, permissions, validation gates, and evidence requirements for an agent system.

Links to:

- Authority Architecture
    
- Decision Authority
    
- Execution Authority
    

---

### 9. Decision Authority

Definition:

> The rules governing which decisions agents may make autonomously.

Links to:

- Human Escalation
    
- Governance Systems
    
- Workflow Design
    

---

### 10. Execution Authority

Definition:

> The rules governing which actions agents may perform through tools or APIs.

Links to:

- Tool Contracts
    
- Authority Packs
    
- Risk Classes
    

---

# Cluster 4 — The Learning Loop

These connect operations to **continuous improvement**.

---

### 11. Context Pack

Definition:

> The structured set of information provided to an agent when making a decision.

Links to:

- Memory Architecture
    
- Authority Constraints
    
- Workflow State
    

---

### 12. Operational Outcome

Definition:

> The measurable result of an agent action or workflow execution.

Links to:

- Execution Logs
    
- Learning Systems
    
- System Evolution

# One Very Important Insight

Notice something interesting:

The center of the graph is **not agents**.

It is **Authority Architecture**.

This is the difference between:

**AI builders**  
vs  
**AI systems architects**.

Builders focus on:

agents  
tools  
models

Architects focus on:

**authority, governance, and operational systems**.
