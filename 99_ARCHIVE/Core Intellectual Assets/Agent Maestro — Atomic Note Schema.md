# Agent Maestro — Atomic Note Schema

Every conceptual note should follow this lightweight structure.

# Title  
  
## Definition  
One clear sentence defining the concept.  
  
## Purpose  
Why this concept exists in agent systems.  
  
## Key Components  
The main elements that make up the concept.  
  
## Relationships  
Links to related keystone concepts.  
  
## Example (Optional)  
Concrete example from a real system.  
  
## Notes (Optional)  
Insights, implications, or open questions.

---

# One Important Rule

Atomic notes should always contain:

Definition  
Purpose  
Relationships

Everything else is optional.

This keeps notes:

- small
    
- linkable
    
- AI-readable
    
- graph-friendly
